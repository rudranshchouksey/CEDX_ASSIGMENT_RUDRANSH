# CEDX System: Production Documentation

This is the global project specification and root documentation for the CEDX Assignment.

---

## 1. Industry & Scope
- **Industry Track:** FinTech (Automated Transaction Auditing)
- **Architectural Tier:** Decoupled Monorepo (Next.js App Router + Python FastAPI)
- **CASE_ID:** `CEDX_ASSIGMENT_RUDRANSH`
- **Dynamic Amendment:** Delivery of high-value payloads (Threshold $T \ge 10,000$) requires a cryptographically validated signature from Role $R$ (`finance_controller`).

---

## 2. Agent Topology
Our backend uses a strict 3-Agent Decoupled Roster to ensure autonomous verification and fallback handling:

- **Orchestrator Agent:** 
  - *Location:* `/backend/src/agents/orchestrator.py`
  - *Contract:* Receives raw `Record` input. Emits a `RoutingDecision`.
  - *Can Call:* `[Worker Agent, Verifier Agent]`
- **Worker Agent:** 
  - *Location:* `/backend/src/agents/worker.py`
  - *Contract:* Receives `RoutingDecision` and parses raw schema strings into validated `ParsedData`.
  - *Can Call:* `[Verifier Agent]`
- **Verifier Agent:**
  - *Location:* `/backend/src/agents/verifier.py`
  - *Contract:* Receives `ParsedData`. Emits a strict `Verdict` (`PASS`, `FAIL`, or `OVERRULE`) returning validation context to the Orchestrator.
  - *Can Call:* `[]` (Terminal Node)

---

## 3. How to Run
To run the decoupled ecosystem locally from a fresh machine, use the integrated Makefile automation:

```bash
# Starts both the FastAPI Backend and Next.js Frontend concurrently
make demo
```
*Frontend runs on `localhost:3000`, Backend runs on `localhost:8000`.*

---

## 4. Controls
- **State Machine Transitions:** Records transition linearly: `draft` -> `in_review` -> `changes_requested` | `approved` | `exception` -> `delivered`.
- **Programmatic Gatekeeping:** Approvals mapped to `delivered` are hard-blocked by programmatic logic requiring Role R validation for payloads breaching Threshold T.
- **Runtime Constraints:** The Orchestrator strictly enforces a token budget per cycle, hard-terminating the span if `cost_usd` exceeds limits to prevent runaway loops.

---

## 5. Planted-Problem Handling
We defensively parse both data-layer errors and agent-layer hallucination faults:
- **Class-A Blocking Codes:** Heuristics immediately flag records with `STALE`, `MISSING_INPUT`, `OUTLIER`, `INJECTION_BLOCKED`, `LOW_CONFIDENCE`, or `UNVERIFIED_ANOMALY`.
- **Agent-Layer Failures:** The Verifier autonomously detects LLM failure states like `AGENT_HALLUCINATION` (emitting false facts), `AGENT_LOOP`, `AGENT_MALFORMED` (JSON schema drift), and `BUDGET_EXCEEDED` (token limits hit).

---

## 6. Generalization
Instead of hardcoding static anomaly thresholds, the Verifier implements mathematical baseline constraints (Median Absolute Deviation / Z-Scores). This allows the agent to correctly catch unseen anomalies on a held-out seed file during grading without relying on rigid, pre-programmed value bands.

---

## 7. LLM/Agent Contract & Eval
- **Evaluation Harness:** Executable via `make eval`.
- **Testing:** We utilize a golden dataset testing suite evaluated by an LLM-as-a-Judge to measure the F1 Score of our Verifier's ability to accurately catch hallucinations vs. true parsing.
- **Contract:** Typed Pydantic schemas enforce rigid JSON boundaries across the wire.

## 9. Amendment
To securely authorize delivery of high-value payloads over Threshold T, a cryptographic SHA256 derivation logic is employed. The server-side FastAPI router `POST /api/review` strictly validates the incoming payload against a salted Role R signature. If the simulation attempts delivery without this payload, the backend returns a `403 Forbidden (Amendment Gate)` and blocks the transition.

---

## 10. AI Usage / Real-vs-Faked
- **AI-Assisted Boilerplate:** The initial Next.js layout shell, Tailwind token compilation, and basic FastAPI routing boilerplates were heavily generated by LLMs to accelerate scaffolding.
- **Human Engineered (Total Ownership):** The cryptographic amendment logic, the multi-agent topology routing, the MAD math heuristic implementations, and the React client-side Live Amendment Interceptor were architected, mapped, and strictly engineered manually to maintain deterministic control.

---

## 11. Tradeoffs & Next Week
- **Tradeoffs:** We opted to duplicate typed interfaces between `/backend/models.py` and `/frontend/types/audit.ts` to maintain true structural decoupling. This requires manual syncs but ensures the backend can be deployed entirely independently of the Next.js runtime.
- **Next Week:** Given another week, we would implement an asynchronous message queue (e.g., Redis/Celery) on the backend to handle the `make demo` pipeline stream dynamically rather than synchronously blocking the FastAPI worker loop.
